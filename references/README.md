# Illustration Package — Reference Images

Side-by-side comparisons proving the illustration pipeline produces
visibly better output than raw `lineTo()` / `lineWidth` rendering.

## Structure

```
references/
├── phase-1-outlines/       ← Geometric test renders (Bars 1-6)
│   ├── bar-01-straight/    ← Each bar gets before/after PNGs
│   ├── bar-02-taper/
│   ├── bar-03-curve/
│   ├── bar-04-s-curve/
│   ├── bar-05-acute-angle/
│   └── bar-06-caps/
├── phase-2-junctions/      ← Junction test renders (Bars 7-11)
│   ├── bar-07-y-junction/
│   ├── bar-10-tree/
│   └── bar-11-trunk-flare/
├── phase-3-marks/          ← Mark strategy renders (Bars 12-21)
│   ├── bar-14-engraving/
│   ├── bar-15-woodcut/
│   └── bar-18-hatch-gradient/
├── phase-4-integration/    ← Full integration renders (Bars 22-24)
│   ├── bar-22-tree-engraving/
│   ├── bar-23-tree-ink-brush/
│   └── bar-24-terrain-hatched/
└── traditions/             ← Symlinks to studies/traditions/ images
    ├── old-plum.jpg → ../../genart/packages/studies/traditions/old-plum/references/Old Plum.jpg
    ├── bamboo-and-rock.jpg → ../../genart/packages/studies/traditions/bamboo-and-rock/references/Bamboo and rock.jpg
    └── schwere-see-v3.png → ../../genart/packages/studies/studies/schwere-see/schwere-see-v3.png
```

## Convention

Each bar directory contains:
- `before.png` — Current rendering method (lineTo / lineWidth)
- `after.png` — Illustration pipeline output
- `notes.md` — Brief pass/fail assessment against QUALITY.md criteria

Images are generated by `examples/render-bars.ts` (to be created).
Tradition references are symlinked, not copied.
